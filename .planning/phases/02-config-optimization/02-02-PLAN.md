Modify the DNS migration section in update.sh (lines 322-381) with these changes:

1. **Add https+local:// detection** (lines 330-332 area). After the `127.0.0.1` check and BEFORE the `dns.adguard-dns.com` grep check, add a new condition:
   ```bash
   elif [[ "$CURRENT_DNS" == "https+local://"* ]]; then
     echo -e "${GREEN}  ✓ DNS -> DoH Local -- сохранено${NC}"
   ```
   This prevents update.sh from re-migrating configs that already have the optimal format.

2. **Change migration target** from old AdGuard DoH to new DoH Local format. In the `else` branch (the actual migration, around lines 333-356), change the `NEW_DNS` JSON from:
   ```json
   {"servers":["https://dns.adguard-dns.com/dns-query",{"address":"1.1.1.1","domains":["geosite:geolocation-!cn"]},"localhost"],...}
   ```
   to:
   ```json
   {"servers":["https+local://1.1.1.1/dns-query","localhost"],"queryStrategy":"UseIPv4","disableCache":false}
   ```

3. **Update the migration log message** from "AdGuard DNS (реклама будет блокироваться)" to "DoH Local (https+local://1.1.1.1)" to reflect the new target.

4. **Update the migration trigger message** from "Миграция на AdGuard DNS" to "Миграция на DoH Local".

5. **Use safe_jq_write instead of raw jq** for the DNS migration (lines 350-354). The current code uses `jq ... > "${CONFIG_FILE}.tmp" && mv ...` which is exactly the anti-pattern Phase 1 eliminated. Replace with the pattern:
   ```bash
   NEW_DNS='{"servers":["https+local://1.1.1.1/dns-query","localhost"],"queryStrategy":"UseIPv4","disableCache":false}'
   if jq --argjson dns "$NEW_DNS" '.dns = $dns' "$CONFIG_FILE" > "${CONFIG_FILE}.tmp" 2>/dev/null && [[ -s "${CONFIG_FILE}.tmp" ]]; then
     mv "${CONFIG_FILE}.tmp" "$CONFIG_FILE"
   ```
   NOTE: update.sh does NOT have safe_jq_write function (it's defined in xrayebator only). Keep the existing inline pattern but ensure it checks `-s` on the temp file before mv (the current code already does this at line 350-354, just update the content).

6. **Keep the QUIC migration unchanged** (lines 361-380). It's still valid and unrelated to Phase 2.

The full DNS section logic should now be:
```
if dns[0] == "127.0.0.1"              -> skip (AdGuard Home, print green)
elif dns[0] starts with "https+local" -> skip (already optimized, print green)
elif has "dns.adguard-dns.com"        -> skip (already has AdGuard DoH, print green -- backward compat)
else                                   -> migrate to DoH Local format
```
